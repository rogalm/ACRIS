USE CASE ID: 6.2<br/><br/>USE CASE DESCRIPTION:<br/>The user wants to search a specific flight. As search criteria the user has to select a date and can optionally enter one or more of the following attributes (or fragments):<br/>- Arrival or departure<br/>- Scheduled time (maybe period of time)<br/>- Airline<br/>- Flight-number<br/>- Destination <br/>- Origin<br/><br/>ACTORS (GOAL):<br/>The user wants to search one or more flights by entering some search criteria.<br/>The application selects all matching flights and displays a list on the screen. <br/><br/>PERFORMANCE GOALS:<br/><br/><br/>PRE CONDITIONS:<br/>The user has an account for the seamless travel platform. <br/>The user has selected a home Airport<br/><br/>POST CONDITIONS:<br/><br/><br/>SCENARIO:<br/>1. The User presses the “Search flight” Button<br/>2. The application displays a search UI with selection items and text fields<br/>3. The user enters search data into the search UI<br/>4. The application selects all matching flights from the database<br/>5. The application displays a list with all matching flights<br/><br/>EXTENSION POINT:<br/><br/><br/>ALTERNATIVE SCENARIO:<br/>If the user hasn’t selected a home airport, the first step is that the application asks the user which Airport should be used for searching flights.<br/><br/>RELATED REQUIREMENTS:<br/><br/><br/>RELATED RULES:<br/><br/><br/>PERFORMANCE MEASURES:<br/><br/><br/>VERSION/HISTORY:<br/>0.1; 19.06.15; LÃ¼cking; added actors<br/>0.2; 02.07.15; Stiffel; Added description & scenario<br/>
