  • Book Service_ActivityGraph
  • Extension-point for optional payment (see below)
  • Repeat steps 2, 3 and 4 until the last form is sent..
  • The application loads the first form-page from the service provider, who offers the service, extends this form with already known data and asks the user to check the filled and fill out the missing fields.
  • The application pre-validates the form and sends it to the service-provider to receive the next booking-step
  • The application provider confirms the booking
  • The application request a fixed booking from the service-provider
  • The application sends the confirmation to the user and displays it
  • The application shows a booking summary to be checked by the user
  • The user checks and fills out the form and presses next step.
  • The user confirms the summary
  • The user leaves the booking process
  • The user presses the button to book a service
  • End
