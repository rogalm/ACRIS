  • The application checks when the booking status was updated
  • The application collects all open bookings (booked services which are not fulfilled)
  • The application requests for each booking the current booking status from the provider if it is older than a configurable time
  • The applications shows a list of all booked and unfulfilled services
  • The service-provider checks the status of the bookings and returns the status to the application
  • The user select the ‘booked services’ menu item in order to get an overview about the booked services
