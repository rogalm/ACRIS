USE CASE ID: 1.2<br/><br/>USE CASE DESCRIPTION:<br/>The user has selected the change password page of the seamless travel platform. He enters his current credentials and the new password. After the system has checked the password policy, the new password is stored for the account.<br/><br/>ACTORS (GOAL):<br/>The user wants to change the password for the Seamless Travel platform.<br/>The application validates the new password against the password policy of the Seamless travel platform.<br/><br/>PERFORMANCE GOALS:<br/><br/><br/>PRE CONDITIONS:<br/>User has an account for the seamless travel platform<br/><br/>POST CONDITIONS:<br/>The password is changed.<br/><br/>SCENARIO:<br/>1. The user selects the change password screen<br/>2. The user enters the login name and old password<br/>3. The user enters the new password<br/>4. The system checks that the password policy is correct <br/>5.
