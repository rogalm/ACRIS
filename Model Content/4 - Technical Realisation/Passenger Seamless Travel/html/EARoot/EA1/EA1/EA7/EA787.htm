USE CASE ID: 5.6<br/><br/>USE CASE DESCRIPTION:<br/>The user wants to search for a specific service. To make it easy to find a service the user can search via a single search field.<br/><br/>ACTORS (GOAL):<br/>The user wants to search services by entering some search criteria.<br/><br/>The application tries to generate a result-list which should be ordered by calculated relevance of the result-items. There for different service fields have different importance for calculating the relevance. E.g. the title is more important than the description.<br/>The result is requested with a length (number of items) and offset (position of the first item to be transferred) to reduce traffic.<br/><br/>PERFORMANCE GOALS:<br/><br/><br/>PRE CONDITIONS:<br/>The user has an account for the seamless travel platform. <br/>The user has selected a default or current airport<br/><br/>POST CONDITIONS:<br/><br/><br/>SCENARIO:<br/>1. The user opens the service overview<br/>2. The application shows a search bar, up to three recommended services and a list of all categories.<br/>3. The user types a search string into the search bar and presses enter<br/>4. The system  searches for all services fitting to the search request<br/>5. The application displays the matching services as list ordered by relevance<br/><br/>EXTENSION POINT:<br/><br/><br/>ALTERNATIVE SCENARIO:<br/>If the user hasn’t selected a default or current airport, the first step is that the application asks the user which Airport should be used.<br/><br/>RELATED REQUIREMENTS:<br/><br/><br/>RELATED RULES:<br/><br/><br/>PERFORMANCE MEASURES:<br/><br/><br/>VERSION/HISTORY:<br/>0.1; 19.06.15; LÃ¼cking; added actors<br/>0.2; 02.07.15; Stiffel; Added scenario<br/>
